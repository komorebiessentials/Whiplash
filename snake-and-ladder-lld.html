02

Clarifying Questions

Three key rule ambiguities — always ask these before designing.

  • ?What if a dice roll overshoots 100?Three common rules: (1) stay put, (2) bounce back, (3) re-roll. Most interviewers say "stay put" — confirm this and model it in Board.getNextPosition().
  • ?Can a snake head and a ladder base be on the same cell?No — validate during board setup and throw IllegalArgumentException. Also: snake tail and ladder top should not be at position 100.
  • ?Starting position — 0 or 1?Players start at 0 (off-board). First dice roll determines entry. Position 100 = win. This is cleaner than starting at 1.
  • ?Can you land on a snake and immediately land on a ladder?In standard rules: no chain reactions. Snake takes you down, that's your final position for the turn.
  • ?How many dice?Usually one. Two dice (sum 2–12) changes balance logic — clarify the interface to allow this.

05

Key Flows

The entire game logic fits in takeTurn() — walk through this step by step in the interview.

1. Get current player
players.get(currentPlayerIndex). Check game.state == IN_PROGRESS, else throw.
2. Roll dice
int roll = dice.roll(). For NormalDice this is random 1–6. For LoadedDice it's the next value in the queue (deterministic for tests).
3. Compute raw new position
int raw = player.position + roll. If raw > board.size → stay (common rule). Raw position before snake/ladder check.
4. Apply board effects
board.getNextPosition(current, roll) returns the final position after snake/ladder resolution. If snake head → tail. If ladder base → top. Else → raw.
5. Determine event type
WIN if to == 100. SNAKE_BITE if snake at raw. LADDER_CLIMB if ladder at raw. NORMAL_MOVE otherwise.
6. Update player position
player.setPosition(to). If WIN → game.state = FINISHED. Don't advance turn on win.
7. Notify observers and advance turn
Emit GameEvent to all listeners. currentPlayerIndex = (currentPlayerIndex + 1) % players.size().

Code

Three key pieces: Dice abstraction, Board.getNextPosition(), and Game.takeTurn().

Dice — strategy for testability
interface Dice {
    int roll();
}

class NormalDice implements Dice {
    private final int sides;
    private final Random random = new Random();

    public NormalDice(int sides) { this.sides = sides; }

    public int roll() {
        return random.nextInt(sides) + 1; // 1 to sides inclusive
    }
}
// For testing — pre-load the sequence of rolls
class LoadedDice implements Dice {
    private final Queue<Integer> rolls;

    public LoadedDice(Integer... sequence) {
        rolls = new LinkedList<>(Arrays.asList(sequence));
    }

    public int roll() {
        if (rolls.isEmpty()) throw new IllegalStateException("No more rolls");
        return rolls.poll();
    }
}

// Usage in test:
Dice dice = new LoadedDice(3, 6, 4, 2); // player will roll 3, 6, 4, 2...
Game game = new GameBuilder().withDice(dice).build();
Board — position resolution
class Board {
    private final int size;
    private final Map<Integer, Integer> snakes;  // head → tail
    private final Map<Integer, Integer> ladders; // base → top

    public Board(int size, Map<Integer,Integer> snakes, Map<Integer,Integer> ladders) {
        this.size    = size;
        this.snakes  = Collections.unmodifiableMap(snakes);
        this.ladders = Collections.unmodifiableMap(ladders);
        validate();
    }

    private void validate() {
        // snake heads and ladder bases must not overlap
        Set<Integer> snakeHeads  = snakes.keySet();
        Set<Integer> ladderBases = ladders.keySet();
        if (!Collections.disjoint(snakeHeads, ladderBases))
            throw new IllegalArgumentException("Snake head and ladder base overlap!");
        // snake tails must be below heads, ladder tops above bases
        snakes.forEach((h, t) -> { if (h <= t) throw new IllegalArgumentException("Invalid snake"); });
        ladders.forEach((b, e) -> { if (b >= e) throw new IllegalArgumentException("Invalid ladder"); });
    }
}
public int getNextPosition(int currentPos, int diceValue) {
    int newPos = currentPos + diceValue;

    // overshoot rule: stay put (most common)
    if (newPos > size) return currentPos;

    // snake: slide down from head to tail
    if (snakes.containsKey(newPos))  return snakes.get(newPos);

    // ladder: climb from base to top
    if (ladders.containsKey(newPos)) return ladders.get(newPos);

    return newPos; // normal move
}

public boolean isSnakeHead(int pos)  { return snakes.containsKey(pos); }
public boolean isLadderBase(int pos) { return ladders.containsKey(pos); }
Game — turn logic
public GameEvent takeTurn() {
    if (state != GameState.IN_PROGRESS)
        throw new IllegalStateException("Game is not in progress");

    Player current = players.get(currentPlayerIndex);
    int roll = dice.roll();
    int from = current.getPosition();
    int raw  = from + roll;
    int to   = board.getNextPosition(from, roll);

    current.setPosition(to);

    // determine what happened
    GameEventType type;
    if (to == board.getSize()) {
        type  = GameEventType.WIN;
        state = GameState.FINISHED;
    } else if (raw <= board.getSize() && board.isSnakeHead(raw)) {
        type = GameEventType.SNAKE_BITE;
    } else if (raw <= board.getSize() && board.isLadderBase(raw)) {
        type = GameEventType.LADDER_CLIMB;
    } else {
        type = GameEventType.NORMAL_MOVE;
    }

    GameEvent event = new GameEvent(type, current, from, to, roll);
    observers.forEach(o -> o.onEvent(event));

    if (state != GameState.FINISHED)
        currentPlayerIndex = (currentPlayerIndex + 1) % players.size();

    return event;
}
class GameBuilder {
    private int boardSize = 100;
    private final Map<Integer,Integer> snakes  = new HashMap<>();
    private final Map<Integer,Integer> ladders = new HashMap<>();
    private final List<Player> players = new ArrayList<>();
    private Dice dice = new NormalDice(6); // sensible default

    public GameBuilder withSnake(int head, int tail)  { snakes.put(head, tail); return this; }
    public GameBuilder withLadder(int base, int top)  { ladders.put(base, top); return this; }
    public GameBuilder withPlayer(String name) {
        players.add(new Player(UUID.randomUUID().toString(), name, 0));
        return this;
    }
    public GameBuilder withDice(Dice dice) { this.dice = dice; return this; }

    public Game build() {
        if (players.isEmpty()) throw new IllegalStateException("Need at least one player");
        Board board = new Board(boardSize, snakes, ladders); // validates in constructor
        return new Game(board, players, dice);
    }
}

Design Patterns

Small problem, but several patterns apply cleanly here.

Behavioral
Strategy
Dice interface — swap NormalDice with LoadedDice for testing without changing Game. This is the most important design decision in the entire problem.
Behavioral
Observer
Game emits GameEvents. UI, logger, or analytics subscribe. Game doesn't know about its observers — clean separation.
Behavioral
State
GameState enum — NOT_STARTED / IN_PROGRESS / FINISHED. takeTurn() rejects calls when FINISHED. Clean guard without conditionals scattered everywhere.
Creational
Builder
GameBuilder chains withSnake/withLadder/withPlayer/withDice. Validates in build(). Far cleaner than a 5-arg constructor.
08

Concurrency

Snake and Ladder is turn-based — single-threaded by nature. Mention this, then note what changes for multiplayer online.

Single-player/local multiplayer: No concurrency issues. Turn-based games are naturally sequential. The only risk is if you expose game state to external reads while takeTurn() runs.
ScenarioRiskSeverityFix
Local multiplayer (same JVM)None — sequential turnsLOWNo action needed
Online multiplayer — networkTwo players submit a turn simultaneouslyHIGHSynchronize takeTurn() + version number (optimistic locking)
Real-time leaderboard readsPlayer position read mid-turnMEDMake Player.position volatile or use an AtomicInteger

→ For online: each turn is a transaction with a version counter. Client sends (turnId, version). Server rejects if version != expected. Classic optimistic locking.

09

Interview Tips

Simpler problem than Parking Lot — differentiate yourself with clean design decisions, not raw complexity.

1
LoadedDice is the most important design insight. When you say "I'll make Dice an interface so tests can inject a deterministic sequence," interviewers immediately know you think about testability. This is the signal they're looking for on an SDE I problem.
2
Separate Board from Game. Board is static — snakes and ladders don't move. Game is dynamic — player positions change. Mixing them makes Game a God class. State this separation explicitly.
3
Board validates itself in the constructor. Snake head and ladder base can't be the same cell. Snake head must be > tail. Ladder top must be > base. Put this in Board's constructor so it's impossible to create an invalid board.
4
Ask about the overshoot rule explicitly. Say "If a player at 97 rolls 6, should they stay at 97, move to 100, or bounce back to 97? I'll implement 'stay put' — let me know if you want a different rule." Shows you know the ambiguity exists.
5
Players start at 0, not 1. Starting at 0 means the first roll always lands them on the board cleanly. Position 100 = win. Avoid off-by-one confusion with a 1-indexed start.
6
Use GameBuilder — it signals maturity. A builder is cleaner than a constructor with 5 arguments. It also lets you add features (multiple dice, custom board sizes) without breaking existing callers.
