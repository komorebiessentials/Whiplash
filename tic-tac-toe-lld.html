Clarifying Questions

Ask to reveal the full scope before designing.

  • ?Fixed 3×3 or configurable N×N?Make it N×N from the start — takes 30 seconds and shows you think generally, not just for the specific case.
  • ?Two human players, or human vs AI?If AI is needed, introduce a Player interface (HumanPlayer, AIPlayer) so Game doesn't know the difference.
  • ?Do we need undo?Store moveHistory as a Stack. Undo = pop last cell, set board[cell] = EMPTY, switch turn.
  • ?Exactly N in a row, or at least K?Standard: exactly N. K-in-a-row (Connect-K) is a more complex generalization — scope it out unless asked.

05

Key Flows

makeMove() is the complete game loop.

1. Validate state and input
Reject if state != IN_PROGRESS. Reject if cell is out of bounds or already occupied.
2. Place symbol
board.makeMove(cell, current.symbol). Increment filledCells.
3. Check winner (O(N) — last move only)
board.checkWinner(cell.row, cell.col) — check only the row, column, and diagonals of the placed cell. No need to scan the whole board.
4. Check draw
If no winner and board.isFull() → DRAW. filledCells == N*N is O(1).
5. Switch turn
currentIndex = 1 - currentIndex. Clean toggle for exactly 2 players.
06

Code

The key insight: check only the last-played row, column, and diagonals — O(N) not O(N²).

Win detection — O(N) per move
// Only check row/col/diagonals of the LAST placed cell — O(N)
public Symbol checkWinner(int row, int col) {
    Symbol s = grid[row][col];
    if (s == Symbol.EMPTY) return Symbol.EMPTY;

    // check row
    if (checkLine(row, 0, row, size-1, s)) return s;
    // check column
    if (checkLine(0, col, size-1, col, s)) return s;
    // main diagonal (only if cell is on it)
    if (row == col) {
        boolean win = true;
        for (int i = 0; i < size; i++) if (grid[i][i] != s) { win = false; break; }
        if (win) return s;
    }
    // anti-diagonal
    if (row + col == size - 1) {
        boolean win = true;
        for (int i = 0; i < size; i++) if (grid[i][size-1-i] != s) { win = false; break; }
        if (win) return s;
    }
    return Symbol.EMPTY;
}

private boolean checkLine(int r1, int c1, int r2, int c2, Symbol s) {
    int dr = Integer.signum(r2-r1), dc = Integer.signum(c2-c1);
    int r = r1, c = c1;
    while (r != r2+dr || c != c2+dc) {
        if (grid[r][c] != s) return false;
        r += dr; c += dc;
    }
    return true;
}
public void makeMove(Cell cell, Symbol symbol) {
    if (!isInBounds(cell)) throw new InvalidMoveException("Out of bounds");
    if (grid[cell.row()][cell.col()] != Symbol.EMPTY)
        throw new InvalidMoveException("Cell already occupied");
    grid[cell.row()][cell.col()] = symbol;
    filledCells++;
}

public boolean isFull() { return filledCells == size * size; } // O(1)
public GameState makeMove(Cell cell) {
    if (state != GameState.IN_PROGRESS) throw new IllegalStateException();

    Player current = players[currentIndex];
    board.makeMove(cell, current.getSymbol()); // throws on invalid

    Symbol winner = board.checkWinner(cell.row(), cell.col());
    if (winner == Symbol.X) state = GameState.X_WON;
    else if (winner == Symbol.O) state = GameState.O_WON;
    else if (board.isFull()) state = GameState.DRAW;
    else currentIndex = 1 - currentIndex; // toggle: 0→1→0→1

    return state;
}

Concurrency

Turn-based, single-threaded. One edge case for online play.

The risk for online games: Two players submit moves simultaneously (race between client and server). The second move arrives before the first is processed.

09

Interview Tips

Tic-Tac-Toe is SDE I — differentiate with optimization and generalization.

1
Check only the last move's row, col, and diagonals — O(N). Don't scan the whole board after every move. Interviewers specifically ask "can you optimize the win check?" — this is the answer.
2
Make it N×N from the start. Say "I'll use size N instead of hardcoding 3 — takes the same effort and covers the general case." Shows you think beyond the literal ask.
3
Toggle with 1 - currentIndex. Cleaner than a conditional or modulo for exactly 2 players. Shows you think about simplicity.
4
filledCells counter = O(1) draw check. Don't count empty cells on the grid every time — increment a counter on each move. isFull() becomes a single comparison.
5
Validate before placing, never after. Check cell bounds and occupancy before board.makeMove(). Don't mutate then check — hard to roll back.
