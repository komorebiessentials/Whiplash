02

Clarifying Questions

Ask to scope the problem — special rules add significant complexity.

  • ?Do we need castling and en passant?Special moves — castling needs "neither king nor rook has moved" state. En passant needs the last move. Both add state to Board/Game.
  • ?Pawn promotion — do we need it?When a pawn reaches rank 8 (or 1), it promotes. Need to ask which piece the player wants.
  • ?Draw conditions — 50-move rule, threefold repetition?These require tracking full game history. Scope them out unless the interviewer specifically asks.
  • ?Do we need a UI or just validation logic?For an interview, just the validation engine. Board state is an 8×8 grid, no rendering needed.
  • ?Starting position — standard or configurable?Assume standard. A custom setup would need a BoardBuilder.

05

Key Flows

The move flow has two gates: piece rules, then king safety. Missing the second gate is the #1 mistake.

1. Player submits (from, to)
Retrieve piece at 'from'. Verify it belongs to currentTurn player.
2. Gate 1 — piece movement rules
Call piece.isValidMove(from, to, board). Each subclass checks its own movement logic (path clear, L-shape, etc.).
3. Gate 2 — king safety (simulate)
Copy the board. Apply the move on the copy. Call isInCheck(currentTurn, copy). If king is in check → move is ILLEGAL.
4. Apply move on real board
board.applyMove(move). Update piece position. Handle captures, promotions, castling (rook also moves).
5. Check opponent state
isInCheck(opponent, board) → check or active. hasLegalMoves(opponent, board) → if none: checkmate (in check) or stalemate.
6. Switch turn, return GameState
currentTurn = currentTurn.opposite(). Return ACTIVE / CHECK / CHECKMATE / STALEMATE.

Code

Implement Rook, Knight, Pawn, and MoveValidator first — these cover all movement archetypes.

Piece hierarchy
abstract class Piece {
    protected Color color;
    protected PieceType type;
    protected Position position;

    public abstract boolean isValidMove(Position from, Position to, Board board);

    protected boolean isEnemy(Position pos, Board board) {
        return board.getPiece(pos).map(p -> p.color != this.color).orElse(false);
    }
    protected boolean isEmpty(Position pos, Board board) {
        return board.getPiece(pos).isEmpty();
    }
    protected boolean canLandOn(Position pos, Board board) {
        return isEmpty(pos, board) || isEnemy(pos, board);
    }
}
class Rook extends Piece {
    private boolean hasMoved = false;

    public boolean isValidMove(Position from, Position to, Board board) {
        // must move in a straight line
        if (from.row != to.row && from.col != to.col) return false;
        return isPathClear(from, to, board) && canLandOn(to, board);
    }

    private boolean isPathClear(Position from, Position to, Board board) {
        int dr = Integer.signum(to.row - from.row);
        int dc = Integer.signum(to.col - from.col);
        int r = from.row + dr, c = from.col + dc;
        while (r != to.row || c != to.col) {
            if (!isEmpty(new Position(r, c), board)) return false;
            r += dr; c += dc;
        }
        return true;
    }
}
class Knight extends Piece {
    public boolean isValidMove(Position from, Position to, Board board) {
        int dr = Math.abs(to.row - from.row);
        int dc = Math.abs(to.col - from.col);
        // L-shape: (2,1) or (1,2) — no path check needed, jumps over pieces
        return ((dr == 2 && dc == 1) || (dr == 1 && dc == 2))
            && canLandOn(to, board);
    }
}
class Pawn extends Piece {
    public boolean isValidMove(Position from, Position to, Board board) {
        int dir       = (color == Color.WHITE) ? 1 : -1;
        int startRow  = (color == Color.WHITE) ? 1 : 6;
        int dr = to.row - from.row;
        int dc = to.col - from.col;

        // forward 1 — must be empty
        if (dc == 0 && dr == dir && isEmpty(to, board)) return true;

        // forward 2 from starting row — both squares must be empty
        if (dc == 0 && dr == 2*dir && from.row == startRow
                && isEmpty(to, board)
                && isEmpty(new Position(from.row + dir, from.col), board))
            return true;

        // diagonal capture — must be enemy piece
        if (Math.abs(dc) == 1 && dr == dir && isEnemy(to, board)) return true;

        return false; // en passant omitted — add with lastMove check
    }
}
MoveValidator — check detection
public boolean isInCheck(Color color, Board board) {
    Position kingPos = board.findKing(color);
    // check if any opponent piece can reach the king
    for (Piece piece : board.getPieces(color.opposite())) {
        if (piece.isValidMove(piece.getPosition(), kingPos, board)) return true;
    }
    return false;
}
public boolean isLegalMove(Move move, Game game) {
    // simulate move on a copy — never mutate real board during validation
    Board copy = game.getBoard().copy();
    copy.applyMove(move);
    // move is legal only if own king is NOT in check after the move
    return !isInCheck(move.getPiece().getColor(), copy);
}
public boolean hasLegalMoves(Color color, Board board, Game game) {
    for (Piece piece : board.getPieces(color)) {
        for (Position target : board.allPositions()) {
            if (!piece.isValidMove(piece.getPosition(), target, board)) continue;
            Move move = new Move(piece.getPosition(), target, piece);
            if (isLegalMove(move, game)) return true; // found at least one legal move
        }
    }
    return false;
}
Game.makeMove — full flow
public GameState makeMove(Position from, Position to) {
    Piece piece = board.getPiece(from)
        .orElseThrow(() -> new InvalidMoveException("No piece at " + from));

    if (piece.getColor() != currentTurn)
        throw new InvalidMoveException("Not your turn");

    // Gate 1: piece movement rules
    if (!piece.isValidMove(from, to, board))
        throw new InvalidMoveException("Illegal move for " + piece.getType());

    Move move = new Move(from, to, piece);

    // Gate 2: king safety (simulate on copy)
    if (!validator.isLegalMove(move, this))
        throw new InvalidMoveException("Move leaves king in check");

    // Apply to real board
    board.applyMove(move);
    moveHistory.add(move);
    currentTurn = currentTurn.opposite();

    // Determine new game state
    boolean opponentInCheck = validator.isInCheck(currentTurn, board);
    boolean opponentHasMoves = validator.hasLegalMoves(currentTurn, board, this);

    if (!opponentHasMoves) {
        state = opponentInCheck ? GameState.CHECKMATE : GameState.STALEMATE;
    } else {
        state = opponentInCheck ? GameState.CHECK : GameState.ACTIVE;
    }
    return state;
}

Design Patterns

Name these explicitly in the interview — vocabulary signals senior-level thinking.

Behavioral
Template Method
Piece defines the validation contract (isValidMove); each subclass implements the step. The "validate, then simulate" sequence is the same for all pieces.
Behavioral
State
GameState (ACTIVE / CHECK / CHECKMATE / STALEMATE). Game behavior depends on state — moves are rejected when state is CHECKMATE.
Behavioral
Command
Move object encapsulates the action. Stored in moveHistory — enables undo, replay, and en passant (lastMove check).
Creational
Factory
PieceFactory.create(type, color, position) — decouples creation from usage. Used when initializing the board or handling pawn promotion.
Structural
Prototype
Board.copy() for check detection simulation. Fast clone of the 8×8 grid and piece references — the core of safe "what-if" evaluation.

Concurrency

Chess is turn-based and single-threaded — but one design risk still matters.

The one real risk: If check detection (which copies the board) and move application (which mutates it) race on the same board object, the copy could be corrupt. Always copy-then-simulate, never share the Board reference across threads.
RiskLocationSeverityFix
Shared board during simulationMoveValidator.isLegalMove()HIGHboard.copy() creates an independent snapshot before applyMove
Concurrent game moves (online)Game.makeMove()MEDSynchronize entire makeMove() — it's atomic from the user's perspective
hasLegalMoves() O(64×16) board copiesMoveValidatorLOWAcceptable for interviews; production uses incremental attack tables

Interview Tips

Chess is deceptively deep — scoping correctly and naming the two-gate insight wins the interview.

1
Two separate validation gates. piece.isValidMove() checks movement rules only. MoveValidator.isLegalMove() simulates the move and checks king safety. They are NOT the same — candidates who conflate them get stuck on check detection.
2
Board.copy() is non-negotiable. Never apply a speculative move to the real board. Always simulate on a copy, check king safety, then (if legal) apply to the real board.
3
Checkmate vs stalemate. Checkmate = in check AND no legal moves. Stalemate = NOT in check AND no legal moves. Both require the same hasLegalMoves() call — only the isInCheck result differs.
4
Knight is special — no path check. Every other sliding piece needs isPathClear(). Knight jumps. This is usually the first thing an interviewer asks to verify you understand.
5
Implement in this order: Rook → Knight → King → Bishop → Queen → Pawn. Rook gives you isPathClear. Knight is the exception. King is 1-step. Bishop reuses isPathClear diagonally. Queen = Rook + Bishop. Pawn is the most special-cased.
6
Scope out castling and en passant first. Say "I'll add en passant after the core is working — it just requires checking board.lastMove." This signals awareness without getting bogged down in edge cases.
