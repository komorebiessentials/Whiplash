02

Clarifying Questions

These shape the design significantly.

  • ?What is the borrowing limit per member?Classic: 5 books max, no books with overdue returns. This is a business rule → goes in a MemberValidator, not Member.
  • ?How is the fine calculated?Flat ₹5/day? First 3 days free? Tiered? Defines the FineCalculator strategy interface.
  • ?How are reservations ordered — FIFO or by priority?FIFO is standard — order by reservedAt timestamp.
  • ?Can a member reserve multiple copies of the same book?Typically no — one active reservation per member per book.
  • ?How are notifications delivered?Email/SMS/in-app. Use NotificationService interface — decouples delivery from library logic.
  • ?Is multi-branch support required?Single branch simplifies. Multi-branch adds LibraryBranch and cross-branch availability queries.

Key Flows

Walk both borrow and return flows in the interview — the return flow with reservation notification is what interviewers listen for.

Borrow Flow

Validate member
Not banned. booksCheckedOut < maxLimit. No overdue items.
Lookup book in catalog
catalog.findByISBN(isbn). Throw BookNotFoundException if absent.
Find available copy
book.getItems().stream().filter(isAvailable).findFirst()
If available: create lending
item.setStatus(BORROWED). item.setDueDate(now+14d). Create BookLending. member.booksCheckedOut++.
If not available: reserve
Create BookReservation(WAITING). Notify member "reserved — we'll alert you".

Return Flow

Lookup active lending
Find BookLending by barcode where returnedAt is null.
Set returnedAt, calc fine
lending.returnedAt = now. fine = fineCalculator.calculateFine(lending). Notify if fine > 0.
member.booksCheckedOut--
Decrement the count so member can borrow again.
Check waiting reservation (FIFO)
Find oldest WAITING reservation for this book. If found: item.status = RESERVED, reservation.status = COMPLETED, notify waiting member.
If no reservation: item = AVAILABLE
Free the copy for walk-in borrows.
06

Code

Focus on the two core library operations and the fine calculator.

Models

class Book {
    private final String ISBN, title, author, category;
    private final List<BookItem> items = new ArrayList<>();

    public Optional<BookItem> findAvailableItem() {
        return items.stream().filter(BookItem::isAvailable).findFirst();
    }
}

class BookItem {
    private final String barcode;
    private Book book;
    private BookStatus status;
    private LocalDate dueDate;

    public boolean isAvailable() { return status == BookStatus.AVAILABLE; }

    // synchronized: prevents two threads from borrowing the same copy
    public synchronized boolean tryReserve() {
        if (!isAvailable()) return false;
        status = BookStatus.BORROWED;
        return true;
    }
}
interface FineCalculator {
    double calculateFine(BookLending lending);
}

class DailyFineCalculator implements FineCalculator {
    private static final double RATE = 5.0; // ₹5 per day

    public double calculateFine(BookLending lending) {
        if (lending.getReturnedAt() == null) return 0;
        long days = ChronoUnit.DAYS.between(
            lending.getDueDate(),
            lending.getReturnedAt().toLocalDate()
        );
        return Math.max(0, days) * RATE;
    }
}

Library — borrowBook + returnBook

public BookLending borrowBook(String memberId, String ISBN) {
    Member member = findMember(memberId);
    validateMember(member);

    Book book = catalog.findByISBN(ISBN);
    if (book == null) throw new BookNotFoundException(ISBN);

    // tryReserve() is synchronized on the BookItem
    Optional<BookItem> available = book.getItems().stream()
        .filter(item -> item.tryReserve())   // atomic check+set
        .findFirst();

    if (available.isEmpty()) {
        reservations.add(new BookReservation(book, member, LocalDateTime.now()));
        notificationService.notify(member, "Reserved. You'll be notified when available.");
        throw new NoBookItemAvailableException(book.getTitle());
    }

    BookItem item = available.get();
    item.setDueDate(LocalDate.now().plusDays(14));

    BookLending lending = new BookLending(item, member, LocalDateTime.now());
    member.incrementCheckedOut();
    lendings.add(lending);
    return lending;
}
public double returnBook(String barcode) {
    BookItem item = findItemByBarcode(barcode);
    BookLending lending = findActiveLending(item);

    lending.setReturnedAt(LocalDateTime.now());
    double fine = fineCalculator.calculateFine(lending);
    if (fine > 0) notificationService.notify(lending.getMember(), "Fine due: ₹" + fine);

    lending.getMember().decrementCheckedOut();

    // Check for waiting reservations — FIFO by reservedAt
    Optional<BookReservation> waiting = reservations.stream()
        .filter(r -> r.getBook().equals(item.getBook())
               && r.getStatus() == ReservationStatus.WAITING)
        .min(Comparator.comparing(BookReservation::getReservedAt));

    if (waiting.isPresent()) {
        item.setStatus(BookStatus.RESERVED);
        waiting.get().setStatus(ReservationStatus.COMPLETED);
        notificationService.notify(waiting.get().getMember(), "Your reserved book is ready!");
    } else {
        item.setStatus(BookStatus.AVAILABLE);
    }
    return fine;
}
private void validateMember(Member member) {
    if (member.getStatus() == AccountStatus.BANNED)
        throw new MemberBannedException(member.getId());

    if (member.getBooksCheckedOut() >= MAX_BOOKS_LIMIT)
        throw new BorrowLimitExceededException("Limit: " + MAX_BOOKS_LIMIT);

    // Check for overdue items — member must clear fines first
    boolean hasOverdue = lendings.stream()
        .anyMatch(l -> l.getMember().equals(member)
               && l.getReturnedAt() == null
               && l.getDueDate().isBefore(LocalDate.now()));

    if (hasOverdue) throw new OverdueItemsException("Return overdue books first");
}

Design Patterns

Name these in the interview.

Creational
Singleton
Library and Catalog are single global instances managing shared state. Use double-checked locking.
Behavioral
Strategy
FineCalculator interface: DailyFineCalculator, TieredFineCalculator, WaivedFineCalculator — swappable without touching Library.
Behavioral
Observer
NotificationService notifies members on overdue and reservation-ready events — decouples delivery channel from library logic.
Structural
Composite
Library → LibraryFloor → Rack → BookItem forms a physical hierarchy. Query availability at any level.
Structural
Facade
Library.borrowBook() hides catalog lookup, member validation, item reservation, and lending creation behind one call.

Concurrency

The last-copy race condition is the critical point here.

Core race condition: Two members request the last available copy simultaneously. Both see it as AVAILABLE. Both create a BookLending. One copy, two borrowers — double booking.
RiskLocationSeverityFix
Double-borrow last copyBookItem.status check+setHIGHsynchronized tryReserve() on BookItem — atomic check+set
Concurrent lending list writesLibrary.lendingsMEDCopyOnWriteArrayList or Collections.synchronizedList
Reservation queue raceLibrary.reservationsMEDConcurrentLinkedQueue for lock-free FIFO
Member booksCheckedOut counterMemberMEDAtomicInteger or synchronized increment/decrement
Catalog index concurrent readsCatalogLOWConcurrentHashMap — reads are lock-free

Advanced: use optimistic locking with a version field on BookItem — compare-and-swap status. Avoids blocking threads while a transaction rolls back.

09

Interview Tips

What separates an SDE I answer from an SDE II answer.

1
Book vs BookItem is the key insight. Every senior interviewer asks for this. Book = title. BookItem = physical copy. Don't conflate them — it shows you've modelled the real domain.
2
Validation is a service, not a method on Member. validateMember() should be in a MemberValidationService, not in Library. Single Responsibility — Member manages its own state; a service enforces business rules.
3
Reservation is FIFO — order by reservedAt. When a book is returned, pick the oldest WAITING reservation. If you use a PriorityQueue ordered by time, this is O(log n) per lookup.
4
The return flow with reservation notification is what interviewers listen for. Most candidates stop after "set status = AVAILABLE". Walk through: check waiting reservation → set RESERVED → notify waiting member → don't omit this.
5
Fine can be 0 even when overdue if there's a grace period. Ask the interviewer. "Is there a grace period?" shows you think about edge cases, not just happy paths.
6
Librarian extends Account, not Member. Librarian has addBook/removeBook/banMember capabilities. Member has borrowBook/returnBook. Both extend Account (shared: id, name, status). Role-based capabilities via inheritance.
